RF Desense Toolbox — User Guide
How to use every tab and feature in this tool
📖 Overview

This tool helps RF engineers measure and track receiver desense — the degradation in radio sensitivity caused by internal noise sources inside a device (CPU, memory, display, audio, charging, etc.).

The workflow has two parts:

  • Individual radio calculators (LoRa, BLE, LTE Low Band, LTE Upper Band, WiFi 2.4G, WiFi 5G, GNSS) — configure the RF chain and compute clean sensitivity and desense for a single measurement.
  • Desense Dashboard — a master table that tracks all radios × all noise sources across a full test campaign, with session save/load so you can resume across multiple lab days.
🔒 Step 1 — Unlock the Tool

Enter the password on the lock screen to access the calculator. The tool is locked by default to prevent accidental edits.

Password: AM2026
⛓ Step 2 — Configure the RF Receive Chain (Friis)

Each radio tab has an RF Chain — Cascaded Noise Figure section at the top. This models your hardware path from antenna to chip using the Friis equation:

F_total = F₁ + (F₂−1)/G₁ + (F₃−1)/(G₁·G₂) + …
  • Each block represents a component: Matching Network, RFFE, LNA, RF SoC/Chip.
  • Enter the NF (dB) and Gain (dB) for each stage. Use negative gain for lossy passives.
  • The Live Block Diagram updates in real time showing NF and gain at each node.
  • The Total NF result feeds directly into the sensitivity calculation below.
📐 Step 3 — Set System Parameters

Below the chain, each tab has a System Parameters card. Fill in:

  • Rx Bandwidth (Hz) — the channel bandwidth of the radio.
  • SNR / C/N₀ — minimum required signal-to-noise ratio for demodulation.
  • KT (dBm/Hz) — thermal noise floor, typically −174 dBm/Hz at room temperature.
  • Antenna Efficiency (dB) — antenna gain/loss relative to isotropic (negative for real antennas).

The tool auto-computes MDS (Minimum Detectable Signal) and Rx Sensitivity (conducted and OTA).

📡 Step 4 — Measure Desense (Spectrum Analyzer)

The Desense Calculation section takes your spectrum analyzer readings:

  • SA RBW (Hz) — Resolution Bandwidth used on the spectrum analyzer.
  • SA Center Frequency (MHz) — tune to the radio's receive frequency.
  • Noise Marker Reading (dBm) — the noise floor reading from the SA marker with the noise source active (e.g. CPU running a stress test).
  • Cable Loss (dB) — loss of the cable between DUT antenna port and SA input.
  • LNA Gain (dB) — if you have an external LNA in the measurement path, enter its gain here.

The tool computes:

  • Noise at DUT antenna — corrected for RBW, cable, and LNA.
  • New MDS — sensitivity with the noise present.
  • Desense (dB) — the degradation. Positive = worse sensitivity.
  • Degraded Rx Sensitivity — conducted and OTA with noise present.
📊 Step 5 — Using the Desense Dashboard

The Dashboard is a master tracking table for a full desense test campaign. Rows = radio bands. Columns = noise sources (Device ON, Video Streaming, CPU Load, etc.).

Workflow per cell:
  1. Click ⚡ Calc on any cell (e.g. LTE Low Band B71 × Video Streaming).
  2. The tool navigates you to the correct radio tab (e.g. LTE Low Band) and shows an amber banner at the top reminding you which measurement is active.
  3. Set up your spectrum analyzer for that noise source and frequency. Enter the SA readings in the Desense Calculation section.
  4. The banner shows the live desense value updating as you type.
  5. When satisfied, click 📥 Push to Dashboard — the desense value is written into the cell and you return to the Dashboard.
  6. The cell shows the desense in color: green (<1 dB), amber (1–3 dB), red (>3 dB). Click ✕ clear to redo.
Noise Source Labels:

The 10 source column labels are editable at the top of the Dashboard. Rename them to match your actual test scenarios (e.g. "4K Video", "BT Music", "Fast Charge").

Device ON column:

The first data column is Device ON — the baseline noise with the device powered on but idle. This is your reference for all other noise sources. It uses the same ⚡ Calc / Push workflow.

💾 Step 6 — Save & Resume Your Session

Desense campaigns take days or weeks. Use the session buttons in the Dashboard header to preserve your work:

  • 💾 Save Session — downloads a .json file containing all entered desense values, device-on readings, and custom source labels. The filename includes a timestamp so you can keep multiple snapshots.
  • 📂 Load Session — opens a file picker. Select a previously saved .json file to restore the full dashboard state exactly as you left it.
  • ⬇ Export CSV — exports the full table (all bands, all sources, all desense values) as a CSV file for use in Excel or reporting tools.

💡 Tips & Best Practices
  • Always measure Device ON first — this is your baseline. All other noise sources are measured relative to this state.
  • Match SA RBW to radio channel BW — use a RBW close to the channel bandwidth for accurate noise floor readings. The tool normalizes the reading to the channel BW automatically.
  • Account for cable loss — always enter the cable + connector loss between the DUT antenna port and the SA input. Even 0.5 dB matters at low signal levels.
  • Save session frequently — click 💾 Save Session after each completed row or noise source. Name your files descriptively (e.g. session_day1_lte_done.json).
  • Desense < 1 dB is generally acceptable — 1–3 dB is marginal and worth investigating, >3 dB is significant and likely to impact real-world performance.
  • Use Export CSV for reporting — the CSV includes all bands, all sources, clean sensitivity, and worst-case desense per row, ready for pasting into test reports.

Hata (Suburban) 150–1500 MHz 1–20 km Suburban / residential LTE suburban coverage Urban PL − 2·[log(f/28)]² − 5.4 dB. Typically 10–15 dB less loss than urban.
Hata (Open) 150–1500 MHz 1–20 km Open rural / farmland Rural LTE, LoRa wide-area Urban PL − 4.78·[log(f)]² + 18.33·log(f) − 40.94 dB. Lowest path loss of the Hata variants.

Model selection tip: Use IEEE 802.11 B/C for short-range indoor/outdoor (WiFi, BLE, LoRa <500 m). Use IEEE S1G for sub-GHz outdoor IoT (LoRa, 868/915 MHz, >500 m). Use Hata for cellular macro-cell coverage (LTE, 150–1500 MHz, 1–20 km).
🧮 Key Formulas
Cascaded NF (Friis)
F = F₁ + (F₂−1)/G₁ + (F₃−1)/(G₁G₂) + …
Noise Floor (MDS)
MDS = KT + 10·log(BW) + NF
Rx Sensitivity
Sens = MDS + SNR
Desense
ΔS = 10·log(1 + P_noise/P_thermal)
Noise at DUT Antenna
P_ant = P_marker − G_LNA + L_cable + 10·log(BW/RBW)
OTA Sensitivity
Sens_OTA = Sens_conducted + Ant_Efficiency
